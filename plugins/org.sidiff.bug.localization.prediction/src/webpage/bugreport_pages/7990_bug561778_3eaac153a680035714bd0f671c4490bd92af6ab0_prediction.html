Code to reproduce: public record Box(T value) { } public class BoxTest { static T test(Box box) { return box.value(); } } Note: The two declaration need to be in a separate file. The statement `return box.value();` will fail to compile, insisting that "Type mismatch: cannot convert from Object to T" This does not occur if both are in the same compilation unit. Moreover, hovering over the `var` restricted type name will display it's type as T: public class BoxTest2 { static T test(Box box) { var val = box.value() return val; } } But the return statement shows the above error. As an other workaround, if the accessor is declared by hand in the record, the test classes will compile fine: public record Box(T value) { T value() { return value; } }

(In reply to Eclipse Genie from comment #2) > New Gerrit change created: https://git.eclipse.org/r/160962 Dev Notes: The issue was that the binary file method_info_t did not have the information that this method is generic. If the test file is in a different file, then this information is read from the binary class and hence the error was signalled. In case, they were in the same file, then the information was from STB which was being compiled and the binary file was not in picture and so did not flag error. In the second compiling scenario of the explicit method definition, the method_info_t was written just like any other method which also added the generic signature info - when the method was not present, a method binding ( SyntheticMethodBinding) is created in which there was this omission and is being corrected by the above gerrit.
