In context of bug 547051 I've checked how one could get this in the IDE and found that References and Declarations search throw exceptions if executed on a class from a modular JDK. Steps to reproduce: Make sure JDK 11 is default and only one Create Java project based on Java 11 Open java.lang.Class type from JDK 11 got to addAll(Collection c, Field[] o) method. Put cursor on "c" variable. Right click and try to References -> (ANY OF, e.g.) Hierarchy Right click and try to Declarations -> (ANY OF, e.g.) Hierarchy An error dialog will be shown, error log has: java.lang.StringIndexOutOfBoundsException: begin 51, end 50, length 72 at java.base/java.lang.String.checkBoundsBeginEnd(String.java:3319) at java.base/java.lang.String.substring(String.java:1874) at org.eclipse.jdt.internal.core.util.HandleFactory.createOpenable(HandleFactory.java:108) at org.eclipse.jdt.internal.core.search.matching.MatchLocator.locateMatches(MatchLocator.java:1474) at org.eclipse.jdt.internal.core.search.JavaSearchParticipant.locateMatches(JavaSearchParticipant.java:112) at org.eclipse.jdt.internal.core.search.BasicSearchEngine.findMatches(BasicSearchEngine.java:250) at org.eclipse.jdt.internal.core.search.BasicSearchEngine.search(BasicSearchEngine.java:595) at org.eclipse.jdt.core.search.SearchEngine.search(SearchEngine.java:670) at org.eclipse.jdt.internal.ui.search.JavaSearchQuery.run(JavaSearchQuery.java:178) at org.eclipse.search2.internal.ui.InternalSearchUI$InternalSearchJob.run(InternalSearchUI.java:95) at org.eclipse.core.internal.jobs.Worker.run(Worker.java:63) That happens independently of patch for bug 547051, but the root cause is similar: the search engine doesn't encode module name properly in the places where search patterns are created, but assumes those in other places. I believe I know where to patch what, will try to push it soon.
