Creation Time 2019-08-20T12:04:24Z
In 4.9 and earlier I was able to add an external class folder to the build path of a java project using a UNC path (i.e. \\foo\...). That doesn't seem to work in version >=4.10. It is possible to add the UNC path to the build path but errors are then shown in the Error Log and nothing appears under "Referenced Libraries" in the package explorer. I.e. the problem only appears for UNC paths. When mapping a drive to the same path and then using the mapped drive path as a build path entry it works as expected. I tested with plain Eclipse installations (no bundles). Also, but this is probably a Subclipse issue, when having the workspace on a UNC path some of the Subclipse operations fails or are inaccessible from the menus. Just thought I'd mention this as it is also related to using a UNC path and seems only to be a problem for the same Eclipse versions as with the above.
