Currently, record components do not have an explicit node, but the required implicit fields and arguments are created. However, given that record components can have separate identity (eg: annotation RECORD_COMPONENT target type), create a new node for record components - Additionally, create a corresponding reference binding.

I sprinkled a few comments into gerrit. As for the issue of annotation duplication: please have a look at method ASTNode.copySE8AnnotationsToType(BlockScope, Binding, Annotation[], boolean) which implements the situation where an annotation could possibly be interpreted as both a declaration annotation and a type annotation. This method additionally handles the case where the parser assumed to find a declaration annotation which according to its @Target is *moved* into the type, with possible re-interpretation wrt qualified names and arrays. I can't off-hand explain how the AnnotationBindings manipulated here contribute to code generation, which needs the source Annotation. copySE8AnnotationsToType seems to use source Annotation only for error reporting? I hope that debugging a case with dual-use annotation (decl & TYPE_USE) will lead you to the relevant locations. In the record case I understand that declaration annotations now apply to two (or more?) declarations, right? Additionally, the same annotation could still be decl & type-use annotation, to add up to 2-dimensional re-use, right? (where the type is shared (not copied) between all declarations).
