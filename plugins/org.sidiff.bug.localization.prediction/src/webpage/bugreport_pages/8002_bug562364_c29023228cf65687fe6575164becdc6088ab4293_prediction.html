The following code is rejected by ECJ: public class X { @SuppressWarnings("preview") public static void main(String args[]) { String result = null; Object obj = "abc"; int i = switch (0) { case 1 -> { yield 1; } default -> { for (int j = 0; !(obj instanceof String s);) { obj = null; } result = s; // 's' should be in scope here System.out.println(result); yield 2; } }; System.out.println(i); } } The problem seems to be that we are resolving the pattern variable in the subscope created by the for statement. But pattern variable should really be in the outer scope.
