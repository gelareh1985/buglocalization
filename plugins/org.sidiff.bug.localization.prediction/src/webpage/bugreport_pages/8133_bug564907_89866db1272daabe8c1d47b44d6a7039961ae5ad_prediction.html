Creation Time 2020-07-14T14:57:00Z
Field and record components are different - hence though from a refactor rename perspective they are same, from the spec as well as even from annotation perspective as mentioned in description they are different and need to be handled differently. I would think that we should have a record component element and then the search can handle that references in a tailored way. If we change the existing meaning for a search for a field, it would work for this case, but in future it may hit us back especially we club the local variable in a compact constructor with field. But if we have IJavaConstants.RECORD_COMPONENT, then the search can "define" the behaviour and return the results including the local variable accordingly.
