Created attachment 277103 Source code demonstrating the issue When a declaration of an enum constant contains a comment like /* something */, the comment prevents the actual Javadoc documentation for that enum constant from being visible in the Javadoc view in Eclipse. With the source --- class Test { public static enum EnumTypeBad { /** First enumerant. */ FIRST /* one */, /** Second enumerant. */ SECOND /* two */, ; } } --- the Javadoc view for FIRST will be fine, but SECOND will have an empty Javadoc in the Eclipse's Javadoc view. Full source that shows the issue is attached, with detailed info what to do to see the problem. The attached source also shows a second EnumTypeGood, which shows that the reason (the only difference) really is the embedded non-Javadoc comment. --- I tried this in the following Eclipse versions (plus a lot of earlier ones): - Version: Photon Release (4.8.0), Build id: 20180619-1200 - Version: 2018-09 (4.9.0), Build id: 20180917-1800
