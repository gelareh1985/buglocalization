If I see it right, the only possibility packageToModule to have no list entry for a given package name is if the JRTUtil.MULTIPLE is read from packageToModule from second thread *before* the first thread that put JRTUtil.MULTIPLE is to packageToModule managed to put the appropriate value to the second packageToModules map. So we have two threads traversing the tree in parallel. But this also means, that we may not only have NP issues, but also problems with list updates, where one thread overrides the list in the map that already set by other thread.
