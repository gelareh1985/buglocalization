(In reply to Till Brychcy from comment #7) > If annotation processing is enabled, MyAnnotation is found as > UnresolvedTypeBinding, and when trying to resolve it, AbortCompilation is > thrown. > > If annotations processing is disabled, MyAnnotation is resolved to a > MissingTypeBinding, but compilation is not aborted. AbortCompilation should not be thrown during completion. It is only thrown if no referenceContext is present, so URB passes environment.root.unitBeingCompleted. The problem is that unitBeingCompleted is reset to null in org.eclipse.jdt.internal.compiler.lookup.LookupEnvironment.completeTypeBindings(CompilationUnitDeclaration, boolean). (In reply to Eclipse Genie from comment #8) > New Gerrit change created: https://git.eclipse.org/r/139790 Patch: Rather than resetting to null, remember previous value and restore it.
