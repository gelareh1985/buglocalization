In https://git.eclipse.org/r/#/c/155140/ I implemented the following approach: If a class is closeable, but not recognized via any black list or white list: * check if it has a no-arg close() method, if so continue as normal * otherwise ask what kind of closeable is the superclass This ensures that subclasses which introduce new GC-resistent resources are excluded from the new logic, while classes which faithfully extend their super are treated in the same way as their super.
