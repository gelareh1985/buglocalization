Creation Time 2013-10-07T10:27:34Z
The current warning for unnecessary casts (and automatic fix) when casting a primitve to an object type is wrong and can lead to very subtle bugs. It is not OK to automatically remove certain primitive -> object casts, as removing the cast would change the semantics. An example: ============================ Integer smallNumber = 42; Integer largeNumber = 500; // this prints: if (smallNumber == 42) System.out.println("42"); // this prints: if (largeNumber == 500) System.out.println("500"); // this prints: if (smallNumber == (Object) 42) System.out.println("42"); // this doesn't print: if (largeNumber == (Object) 500) System.out.println("500"); ============================ Removing the casts on "(Object) 42" and "(Object) 500" will change the behaviour of the programme. - With the cast, "largeNumber == (Object) 500" will auto-box largeNumber and thus perform an object identity comparison. The two objects aren't identical (as the Integer.valueOf() cache caches values < 128 by default) - Without the cast, the compiler performs auto-unboxing on largeNumber and compares int values
