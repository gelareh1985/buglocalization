Bug occurs only when type annotations are handled (e.g., by enabling annotation processing, or null type annotations). Back in https://git.eclipse.org/c/jdt/eclipse.jdt.core.git/commit/?id=3b7e9a0f1d7c1bcba3808ca366020fd264fda3dd Srikanth added this in the new copy constructor: // this.scope = prototype.scope; // Will defeat CompilationUnitDeclaration.cleanUp(TypeDeclaration) && CompilationUnitDeclaration.cleanUp(), so not copied, not an issue for JSR 308. The last part turns out to be wrong. Not having a scope can be an issue for an annotated variant of a local type.
