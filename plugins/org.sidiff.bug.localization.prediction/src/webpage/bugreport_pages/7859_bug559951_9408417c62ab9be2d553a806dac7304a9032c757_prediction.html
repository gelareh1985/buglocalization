Creation Time 2020-02-09T15:47:48Z
While fiddling with some (admittedly a bit esoteric) typing, I think I have found a bug in ecj. It seems that even though the compiler correctly infers a multiple bounded type, and accepts accessing members of any of the types of the multiple bounded type, when the multiple bounded type appears as a lambda argument, only the first of the listed types can have methods invoked during runtime. As this is a little involved to explain with text only, I have prepared small project which reproduces the problem: https://github.com/runeflobakk/broken-method-dispatch (more details in the readme) src/main/java contains a small example domain model, where in particular WithMessageRecipients contains a method with the following signature: List getMessageRecipients(); src/test/java contains a test which accesses methods on both the WithName and WithEmailAddress interfaces on elements of the list returned from the getMessageRecipients() method. The one using a for loop assigning each element to an inferred var type works fine, but when the elements appears as lambda arguments using a Stream, the invocation fails at runtime with a NoSuchMethodError. The code both compiles and executes with not problems with the OpenJDK compiler.
