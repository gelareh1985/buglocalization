The spec under 8.1.6 says this: "A sealed class C without an explicitly declared permits clause, has an implicitly declared permits clause that lists as permitted subclasses all the classes in the same compilation unit (7.3) as C that declare C as their direct superclass." As per this, the following code should be allowed provided both are in the same CU, let's say X.java sealed class X {} non-sealed class Y extends X {} But we get this error: The type Y extending a sealed class X should be a permitted subtype of X
