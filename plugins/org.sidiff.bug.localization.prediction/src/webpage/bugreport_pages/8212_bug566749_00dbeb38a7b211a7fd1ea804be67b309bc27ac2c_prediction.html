Creation Time 2020-09-08T03:58:12Z
In certain cases, the ModuleFinder reports an error via an IllegalArgumentException, for e.g, module name not matching the folder it is residing in. However, EclipseCompilerImpl simply catches the exception and does nothing about it. The corresponding line no is EclipseCompilerImpl:615 We should catch IllegalStateException and log a warning and for everything else, we should report to the diagnostic listener.
