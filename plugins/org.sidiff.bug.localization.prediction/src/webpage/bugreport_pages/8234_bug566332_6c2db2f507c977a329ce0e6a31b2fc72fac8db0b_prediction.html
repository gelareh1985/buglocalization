The following code which worked well using javac of Oracle Java SE 11 (11.0,8), however doesn't work in Eclipse 2020-06. public class Tester { public static void main(String[] args) { String switchVar = "abc"; final String caseStr = true ? "abc" : "def"; switch (switchVar) { case caseStr: System.out.println("Test passed"); } } } It worked well with javac in command line which prints: Test passed However, it showed an error in Eclipse: Exception in thread "main" java.lang.Error: Unresolved compilation problem: case expressions must be constant expressions According to JLS ยง4.12.4 and JLS ยง15.28, the String type can be a final variable and ternary operator can also be counted as constant expression. So using final string variable with ternary operator is fine. So I think this is a bug of eclipse.
