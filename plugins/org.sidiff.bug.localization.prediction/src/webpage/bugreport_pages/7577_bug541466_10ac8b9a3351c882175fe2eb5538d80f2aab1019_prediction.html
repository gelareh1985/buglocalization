Relates issues: https://github.com/redhat-developer/vscode-java/issues/693 https://github.com/micronaut-projects/micronaut-test/issues/5 If an annotation references a class not on the class path Eclipse's APT implementation fails with the below exception. The issue https://github.com/redhat-developer/vscode-java/issues/693 has an example with steps to reproduce. This behaviour is different to how both javac's annotation processing and Java runtime behave and is bug in the Eclipse implementation of APT.
