Creation Time 2020-08-14T16:54:41Z
After review of search results, I believe this is a different issue to or an edge case of previous issues. When linking to a method in Javadoc, the UI will not linkify or follow the method reference if there is a Java annotation preceding the javadoc block. For example, the following will work: public class foo { public void a() { } /** * @see #a() */ public void b() { } } The method a() will link from the javadoc from method b(). However, the following will break: public class foo { public void a() { } @Test /** * @see #a() */ public void b() { } } With the annotation preceding the comment block, which appears to be valid syntax, the method reference will not be clickable. This happens with native annotations (e.g. @Deprecated), imported annotations (e.g. @org.junit.Test), and also Eclipse parser annotations (e.g. @SuppressWarnings) Note that *class* links *will* work in this case. Workaround: put annotations below javadoc block. (Note too that if you already have an annotation preceding the javadoc block, Eclipse autocomplete options for adding method annotations (e.g. method-level @SuppressWarnings) will join the existing annotation above the javadoc block. Otherwise it will place in the "safe" position.)
