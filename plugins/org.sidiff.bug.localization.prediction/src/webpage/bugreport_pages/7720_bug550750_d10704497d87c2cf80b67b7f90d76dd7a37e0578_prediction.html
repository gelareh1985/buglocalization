Ref https://mail.openjdk.java.net/pipermail/amber-spec-experts/2019-December/001859.html regarding how to identify records - this is not a grammar change but change in JVM and in our compiler.ast creation we follow this convention for TypeDeclaration (ACC_ENUM, ACC_INTERFACE, etc) - This cannot be done anymore for Records since there is no ACC_RECORD - The current checked in needs to be modified to incorporate this change - essentially this means we cannot just look at the modifiers and decide whether this is a record or not - So at language level proposing to add an TypeDeclaration.isRecord() which needs to be checked if the modifiers flag it as a class.
