Why was the API not evolved in the usual way? > > > > IMethodBinding is tagged as @noimplement since 2008. > > Does this mean that is is not API? Is this described somewhere? It *is* API, but it is supposed not to be *implemented* by clients. Means, you can use it in code, cast etc to IMethodBinding, but we do not expect anyone create a class *outside of jdt.core* bundle implementing it. See IMethodBinding.java line 38: @noimplement This interface is not intended to be implemented by clients. Also see https://wiki.eclipse.org/API_Javadoc_tags
