Creation Time 2019-10-15T13:02:05Z
The problem happens only in the use of Switch Statement. In the case of Switch Expression this is handled correctly. In this case the Created ASTNode for the Switch Expression is null, and then the code gets pasted and all Quick Fixes are available. In the case of Switch Statement, this is not handled and we get Exceptions mentioned.It would be good to handle Switch Statements in a similar way to the Switch Expression so that users can use the paste functionality and the quick fixes. Suggestion: In the AST the Switch Statement could be probably replaced with an empty statement so that the paste action is complete. Moving this bug to JDT Core.
