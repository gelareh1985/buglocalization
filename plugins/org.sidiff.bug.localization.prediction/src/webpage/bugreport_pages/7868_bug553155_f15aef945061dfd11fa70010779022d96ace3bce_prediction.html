Creation Time 2020-01-14T21:41:55Z
Looks like we'll need a lot of new formatter settings: - indent -> declarations within record declaration -> brace positions - record declaration - record constructor declaration - parenthesis position -> record declaration -> whitespace -> records - before opening parenthesis - after opening parenthesis - before closing parenthesis - between empty parenthesis - before comma - after comma -> record constructor declaration - before comma in 'throws' clause - after comma in 'throws' clause - before opening brace -> new lines -> keep braced code on one line - record declaration - record constructor declaration -> line wrapping -> wrapping settings -> record declaration - record components - 'implements' clause - 'throws' clause in record constructor declaration It's tempting to use existing settings for classes/constructors, but since we did't use this kind of approach for enums/annotation types, it would look inconsistent. We can at least set these new preferences to the values of their existing counterparts on workspace migration to save on clicking.
