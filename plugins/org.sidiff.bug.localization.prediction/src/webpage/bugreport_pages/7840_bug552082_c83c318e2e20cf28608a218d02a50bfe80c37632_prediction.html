Creation Time 2020-01-30T18:19:05Z
I'll hijack this bug to implement the spec change in https://bugs.openjdk.java.net/browse/JDK-8231435 which has been resolved for Java 14: > Proposal: > > If an annotation of type java.lang.annotation.Target is not present on the > declaration of an annotation type T, then T is applicable in all nine > declaration contexts and all 16 type contexts. This subsumes the issue in comment 0. This change made me look also at this in existing JLS §9.7.4: "If the annotation's type is applicable in the declaration context corresponding to the declaration and in type contexts, then the annotation is deemed to apply to both the declaration and the type which is closest to the annotation." I don't see this dual applicability represented in the class files generated by javac nor ecj. Just mentioning as am NOT going to fix this, i.e., I will not invest efforts to become incompatible with javac in this case.
