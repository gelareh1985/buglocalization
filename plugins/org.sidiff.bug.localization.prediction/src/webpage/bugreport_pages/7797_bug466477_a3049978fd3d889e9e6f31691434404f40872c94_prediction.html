With null annotations enabled: //--- import org.eclipse.jdt.annotation.*; public interface SuperI { void test(@NonNull String s); } //--- public class Base { public void test(String s) { } } //--- public class Custom extends Base implements SuperI { } //--- Warning is raised as: ---------- 1. WARNING in /tmp/Custom.java (at line 1) public class Custom extends Base implements SuperI { ^^^^^^ Missing non-null annotation: inherited method from SuperI specifies this parameter as @NonNull ---------- How does the user know, *which* method is to blame? This message is insufficient for those cases where we complain against an inherited concrete method not conforming to a contract inherited from another super type.
