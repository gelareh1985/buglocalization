Created attachment 92057 Example of formatting issue with GWT code I use MyEclipse and had originally contacted them about this bug, but their development team said that it is a bug within Eclipse and to file the bug here. For consistency, here is a link to that original post: http://www.myeclipseide.com/index.php?name=PNphpBB2&file=viewtopic&p=90358 There is a formatting bug when Eclipse formats GWT native-style code. The setting is: Window / Preferences / Java / Code Style / Formatter / Edit / Indentation - "Declarations within class body" If this is on (checked), everything formats correctly except GWT native methods. Every format (which for my team is also set up to occur during save), it inserts a single space at the beginning of each GWT (non-Java) line. If the setting is off (unchecked), every format/save deletes a single sapce from each of the line(s). It should leave it alone, or format it according to the other code rules. Please see the attached file for the example. This is just a small example. Imagine a very large file. This affects us when checking in modified code, because the differences show up when doing a compare at check-in time. I would really like to see this resolved in the patch release. Any chance of that happening? Mike Viens
