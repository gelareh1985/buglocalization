Generate getter/setter creates unnecessary blank line if format setting "Blank Lines within type declarations" > "After last member declaration" is > 0. I think the "After last member declaration" (org.eclipse.jdt.core.formatter.blank_lines_after_last_class_body_declaration) should not have any effect inside the method. To reproduce: 1. Set Formatting to Eclipse [built-in] 2. Change "After last member declaration" to 1 3. Generate a getter, e.g. using the content assist The result is: > /** > * @return the field > */ > public String getField() { > return this.field; > > } When executing the format action the blank line is (correctly) removed.
