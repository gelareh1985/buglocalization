The patch takes care of the reconciler issue. However, we need to discuss what the model API should look like. Especially, I would like to hear from the UI team on what they need. We have the option of letting IField to represent record component and optionally add things like getAccessor() etc. on the IField itself. On the other hand, We can create IRecordComponent to represent the record component properly. This would mean that we perhaps need IType#getRecordComponent() and probably have to alter IType#getFields() to include or not include record components. Considering that IRecordComponent won't be much different from IField, a totally new element is an overkill. Kalyan, what do you think? Also, I see that code assist still resolves the record components to fields. If we take the approach of a new element kind, we might have to touch quite a few places.
