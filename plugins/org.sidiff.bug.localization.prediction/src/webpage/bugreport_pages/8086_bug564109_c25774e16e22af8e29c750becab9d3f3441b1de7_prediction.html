As of now, the following changes have been made to the javax APIs: 1. Two new modifiers SEALED and NON-SEALED have been added to javax.lang.model.element.Modifier. 2. New preview method getPermittedSubclasses() has been added to TypeElement. JDT should support these for Java 15.

(In reply to Eclipse Genie from comment #1) > New Gerrit change created: https://git.eclipse.org/r/164590 The patch disables all RecordElementsTests#test*Javac() tests for the time being. Looks like Javac has changed its behavior in how it treats annotations on record components. For example, the following code is rejected with three errors in JDK 15 but the code was allowed in JDK 14. public record Point(@MyAnnot int comp_, // rejected by javac 15 @MyAnnot3 int comp3_, // rejected by javac 15 @MyAnnot4 int comp4_ // rejected by javac 15 ) { } @Target({ElementType.RECORD_COMPONENT, ElementType.TYPE}) @Retention(RetentionPolicy.RUNTIME) @interface MyAnnot {} @Target({ElementType.METHOD}) @Retention(RetentionPolicy.RUNTIME) @interface MyAnnot3 {} @Target({ElementType.FIELD}) @Retention(RetentionPolicy.RUNTIME) @interface MyAnnot4 {} However, the spec still maintains that these should be allowed. Perhaps things are in a flux and might change. NOTE: Do not close this bug unless the tests are investigated and pass again.
