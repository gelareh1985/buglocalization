Creation Time 2020-09-29T11:42:07Z
In our application having >500 java modules, in some larger java file the content assist takes >3 seconds to show the methods of another larger java class. In this situation, the invocation count of NameLookup.isPackage() is >600k. It turns out that isPackage() does String.join(".") the package name String array for the moduleRoot.getPackageFragment() API, which in turn does split up that name using Util.getTrimmedSimpleNames() back again. Avoiding that join+split can save >0.5 seconds in above situation, where it is ~20 percent. What do you think?
