Creation Time 2020-06-09T20:56:38Z
According to the JLS §8.10.4 [1] a non-canonical constructor must call an other constructor from the record: > If a record type R declares constructors whose signatures are not override-equivalent to the derived constructor signature of R, then each constructor (if any) must satisfy all of the following; otherwise a compile-time error occurs: > > * The constructor body must start with an explicit constructor invocation of another constructor of the record type R (including possibly the canonical constructor). ejc doesn't check this. [1]: https://docs.oracle.com/en/java/javase/14/docs/specs/records-jls.html#jls-8.10.4
