Creation Time 2020-04-27T05:32:59Z
Initial idea at the early stage of preview was to create a separate RecordDeclaration (compiler.ast) node - However, at this stage, based on review comments of bug 562439 (Thanks Steephan) it is deemed that we can have it as a separate case of TypeDeclaration - similar to that of handling Enum, although we don't have an equivalent of ACC_ENUM. This would also help in streamlining the code for bug 562439 and more importantly to have a uniform treatment of all type declarations by representing in an all encompassing node TypeDeclaration.
