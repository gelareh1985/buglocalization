Bug Fix
Bugfix Commit Bug 567512 - Concurrent modification on clean build with xtext builder Race conditions can occur when JavaModelManager.PerProjectInfo.indexingSecondaryCache is accessed, e.g. when deleting Java sources with secondary types, while the indexing of those secondary types is stil ongoing. In particular, its possible that a secondary type is being added to a cache "at the same time" as a secondary type is being removed from the cache, or while the cache is being iterated over due to a merge operation. Both can result in concurrent modification exceptions. This change adds extra synchronization to prevent concurrent container access and modifications; the cache is changed to a synchronized map and iterating over it is synchronized on the map.
