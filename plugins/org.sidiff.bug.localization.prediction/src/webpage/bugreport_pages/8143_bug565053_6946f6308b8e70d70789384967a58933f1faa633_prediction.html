Creation Time 2020-07-08T12:28:18Z
I have Line Wrapping Policy: Wrap all arguments, every element on a new line for function calls/arguments and function calls/qualified object allocation arguments. It seems that when one of the argument's comma is just after the line wrap, the wrapping algorithm errors out and stops wrapping completely. Consider the following example: class Example { private static final List SUPPORTED_THINGS = Arrays.asList( new Thing(Cat, "alskfjalskjfalskfjlkasjf", "asasgf", "asfasfa"), new Thing(Dog, "asfasfafwqfwfasf", "qqwgfagasgasgasgag", "asasg"), new Thing(Bird, "agwgewtagagadgd", "qweasgasdgasasdasd", "asfasfasf"), new Thing(Elephant, "asgfasgwqrqwa", "agsasgasgasg", "asasgasgasg"), new Thing(Crocodile, "asgasgasgasgasgasg", "asgasgasgasgasgasfasfgasf", "adad"), new Thing(Bee, "asgasgasgasgasgasg", "asgasgasgasgasgasg", "asgasgasgasg") ); } For a line width of 86 this is fine, but reduce this to 85 and "Bee" will be in the same line as "Crocodile". Reduce it to 75 and everything after "Bird" will be in the same line as "Bird". This behavior can be observed in the formatting preview as well.
