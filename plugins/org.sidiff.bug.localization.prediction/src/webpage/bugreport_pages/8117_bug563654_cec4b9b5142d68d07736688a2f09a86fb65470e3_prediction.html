Creation Time 2020-05-28T07:25:38Z
Compiling the following source with the JSR-199 EclipseCompilerImpl reports the boxing warning through the DiagnosticListener even though it should be suppressed: package p; @SuppressWarnings("unused") public class SuppressTest { private String unused="testUnused"; @SuppressWarnings("boxing") public Long get(long l) { Long result = l * 2; return result; } } Compiling the following source with the JSR-199 EclipseCompilerImpl reports the boxing warning twice through the DiagnosticListener: package p; @SuppressWarnings("unused") public class SuppressTest { private String unused="testUnused"; public Long get(long l) { Long result = l * 2; return result; } } The BatchCompiler does it correctly: doesn't report any warning in the first case, reports the boxing warning once in the second case. Of course, include -warn:+unused,boxing when compiling. Since plexus-compiler has been switched to call the JSR-199 EclipseCompiler (because of otherwise missing extraProblems), this problem now surfaces in maven builds using plexus-compiler 2.8.6 or newer. Therefore setting this to "major". Compare also bug 469725.
