The changes from bug 543873 and bug 543996 independently introduced new constants in JavadocTagConstants. Unfortunately, there is now a conflict on value 21 used by: public static final int TAG_SYSTEM_PROPERTY_VALUE=21; public static final int TAG_USES_VALUE=21; Interestingly, the comments of TAG_NAMES try to resolve this conflict: TAG_SYSTEM_PROPERTY, /* 21 */ TAG_USES, /* 22 */ but obviously, this has no effect :) Another mistake can be found in the comments of JAVADOC_TAG_TYPE: TAG_TYPE_INLINE, // TAG_DOC_ROOT_VALUE = 20; TAG_TYPE_INLINE, // TAG_DOC_ROOT_VALUE = 21 TAG_TYPE_BLOCK, // TAG_HIDDEN_VALUE = 22; I believe all these arrays and constants must be carefully synchronized.
