(In reply to Jay Arthanareeswaran from comment #0) > To my understanding, the PublicScanner should pretty much have what we have > in the internal Scanner and the ITerminalSymbols should also include the new > token for text block. IIRC, pre Java 8, PublicScanner was a verbatim copy of Scanner, with the only difference to use a different set of token constants. Already the VanguardParser needed for classifying Java 8's '->' and '@' tokens was never added to PublicScanner. Then for Java 9 we had to add a Caveat that the scanner is not useful for module-info.java. Swith expression's 'yield' token shares the same fate. Just listing these cases so we have a reference for stuff that we don't plan to copy to PublicScanner. Message is: scanning Java code is becoming a less and less fruitful task. Nothing we can do about it. Still, I wonder if we should copy getCurrentTextBlock() to PublicScanner for the benefit of bug 552331.
