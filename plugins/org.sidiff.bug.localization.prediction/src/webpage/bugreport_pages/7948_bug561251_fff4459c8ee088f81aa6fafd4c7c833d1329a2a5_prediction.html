Creation Time 2020-03-19T10:43:28Z
This was originally reported via bug 561239: 1. generation of multi-line strings is WRONG, starts with a spurious newline The code looks like private static final String MESSAGE = """ This is a multi-line message that is super- exciting!"""; ... System.out.println("Playing with recent JDK features:"); System.out.println(MESSAGE); and the generated output is: Playing with recent JDK features: This is a multi-line message that is super- exciting! that blank line is there is NOT CORRECT. Investigation shows that in case of CR+LF, we treat the LF as part of the text block literal. This is wrong.
