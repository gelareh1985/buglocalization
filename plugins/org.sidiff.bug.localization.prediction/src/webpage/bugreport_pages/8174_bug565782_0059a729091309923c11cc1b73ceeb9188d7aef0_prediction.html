sealed interface I permits X {} enum X implements I { ONE {}; } As per the changes in section 8.9 of JLS 360 Sealed (preview 1), an enum declaration is implicitly sealed if it contains at least one enum constant that has a class body; which is achieved by marking the anonymous classes implicitly declared by the enum constants that have a class body as the permitted subclasses.

We are setting the permitted types as soon as the anonymous types are available and hence we cannot move this earlier. However, the annotation processor, I believe, doesn't require to know the permitted types - it just requires to know whether they are present or not, because it is looking only at modifiers. This can be determined earlier and the modifiers updated accordingly. Have uploaded a patch on the same gerrit with the crux of the patch being this fix.
