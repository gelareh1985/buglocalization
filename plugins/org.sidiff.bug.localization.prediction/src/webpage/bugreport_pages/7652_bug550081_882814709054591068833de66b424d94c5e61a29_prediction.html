Creation Time 2019-08-14T19:51:52Z
I was running a runtime workbench with PDE tests in my host. In the runtime workbench I installed a new JRE and was greeted by You selected a JRE that this version of Eclipse JDT does not yet support fully. Some of the features may not work as expected. when only 1.8 and 9 were installed. During debugging I found that the "latest" version in getAllVersions() was indeed "1.3". How so?? Class org.eclipse.pde.ui.tests.ee.EnvironmentAnalyzerDelegate had succeeded in re-ordering java versions (reversing all but the first 3). JDT/Core should make this impossible (at which point the PDE test probably needs adjustment).
