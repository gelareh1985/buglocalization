org.eclipse.jdt.internal.compiler.lookup.ParameterizedTypeBinding is created from a ReferenceBinding (usually org.eclipse.jdt.internal.compiler.lookup.SourceTypeBinding or BinaryTypeBinding). PTB copies fields, methods, etc. so that the resolved type argument(s) can be applied. And it delegates to its ReferenceBinding for other things, like annotation tag bits. Annotations are not delegated to ReferenceBinding and are not saved onto PTB when it is created. So, if you have: @foo.bar.Baz abstract class A {} class C extends A {} The ParameterizedTypeBinding created for "A" has reference to "A" in its "type" field. But asking the PTB for its annotations returns empty array because "type" is not consulted for annotations. In this case, asking the superclass of C if it has "Baz" annotation returns false. Suggested fix, add delegating method in ParameterizedTypeBinding above getAnnotationTagBits(): @Override public AnnotationBinding[] getAnnotations() { return this.type.getAnnotations(); } @Override public long getAnnotationTagBits() { return this.type.getAnnotationTagBits(); }
