Creation Time 2019-03-05T16:36:53Z
Currently the Java Formatter knows 2 ways to format annotations: 1. Put each annotation on a separate line. 2. Put all annotations for an element on the same line in front of that element. Nowadays in Enterprise applications with Spring, Lombok, Swagger, etc. you can easily have 5 - 10 annotations on a single Java element, most of them fairly short. This means that option 1 wastes a lot of lines, and option 2 is useless because the annotations can take up too much space (5-10 annotations at 5-10 characters can become 100 characters at the beginning of the line). What I need is an option to just leave annotations alone so that I can put several of them on the same line as I see fit. Currently that is not possible if I want to use the Java Formatter. Even better would be some rules as which annotations should go on the same line (could be driven by package name, or just manual grouping).

Creation Time 2019-03-06T22:35:32Z
Defining formatting behavior per annotation name or package seems too complex, but a pretty simple and flexible solution would be to create a wrapping setting for multiple annotations. Then it can be set to never wrap (keep all in one line), keep each one in a separate line, or wrap where necessary (which roughly means "leave them alone" if "never join already wrapped lines" is on). Only in case of annotations it doesn't make sense to indent elements, so the indentation control would be disabled. And I'm not sure whether there should be multiple settings like for new lines (for types, methods, fields, etc) or just one setting to catch them all.
Creation Time 2019-03-07T01:38:37Z
In my opinion, formatting multiple annotations to be on a single line doesn't work in so many practical cases that it doesn't seem useful as a global setting. It could be great though if you just have one or maybe 2 annotations.
