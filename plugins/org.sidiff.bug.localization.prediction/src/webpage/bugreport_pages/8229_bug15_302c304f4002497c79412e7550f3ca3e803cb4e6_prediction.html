In support of some of the VCM CVS work we need an SSH implementation that we can use. There are two basic options, Java client implementation, native C implementation. The native C story has the standard drawbacks of having to spawn an external process and manage the standard in/standard out. The Java approach would be more closely integrated but may suffer in performance and flexibility due to Java's inferiour socket implementation. NOTES: JM (18/04/2001 5:21:19 PM) There are also licensing issues. many of the public implemenations are GPL. This may or may not be an issue. A good place to start looking is http://www.openssh.org. The SSH Plugin Java implementation is likely to be the most promising of the Java guys. To test/investigate this area we need an SSH enabled Linux box. Moving to JEP for investigation JEP (4/24/01 3:49:53 PM) Here is what I've done so far: 1. Installed openssh-2.5.2p2 (includes linux ssh client and server) on dj. 2. Logged on to dj from dj using linux ssh client. 3. Installed the Java Telnet Application 2.0 (includes java ssh client plugin) on pico1. 4. Logged on to dj from pico1 using the JTA ssh client plugin.
