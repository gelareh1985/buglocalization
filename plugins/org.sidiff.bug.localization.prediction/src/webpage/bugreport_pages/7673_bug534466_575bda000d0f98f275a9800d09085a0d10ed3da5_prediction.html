Creation Time 2018-05-08T14:18:54Z
See https://stackoverflow.com/questions/50149194/reference-to-method-is-ambiguous-method-reference-vs-lambda From two overloads: public static Test define(FuncN zipperFunc,TODO... tasks) public static Test define(TODO... tasks) ecj picks the former when invoked like this Test.define(State::mergeStates,()->true); where mergeStates is a varargs method. According to JLS §15.12.2.4. this argument should not be considered during applicability test, so both methods define() are applicable. Since none is more specific than the other compilation should fail. By contrast, ecj happily compiles the program. Reason: ReferenceExpression.isCompatibleWith(TypeBinding, Scope) doesn't check pertinence to applicability, nor does isBoxingCompatibleWith() perform any real checks.
