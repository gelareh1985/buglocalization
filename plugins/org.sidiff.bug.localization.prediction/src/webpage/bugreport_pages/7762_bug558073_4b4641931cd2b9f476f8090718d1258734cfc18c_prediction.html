Consider the following code: @SuppressWarnings("unused") public class Cls { public static int i = 0; public static void main(String argv[]) { byte v1 = (byte)0; char v2 = ' '; var v = switch(i+1){ case 1 -> v2; case 5 -> v1; default -> v2; }; System.out.println(typeName(v)); // should print, but prints char } private static String typeName(byte arg){ return "byte"; } private static String typeName(char arg){ return "char"; } private static String typeName(int arg){ return "int"; } } The relevant part of the spec under 5.6 says: "Otherwise, if any expression is of type char, and every other expression is either of type char or a constant expression of type int with a value that is representable in the type char, then T is char and the int expressions undergo narrowing primitive conversion to char." We know a byte to char is lossy and disallowed. Hence, the following from the spec becomes relevant: "Otherwise, T is int and all the expressions that are not of type int undergo widening primitive conversion to int."
