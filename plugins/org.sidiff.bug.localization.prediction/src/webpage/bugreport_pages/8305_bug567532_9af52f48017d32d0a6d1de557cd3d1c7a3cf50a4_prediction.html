This seems to be another Module System issue. 1. Create Java 9 project 2. Modify .classpath to contain: 3. Enable builder debugging The output says: > Successfully read state for abc > JavaBuilder: New location: Classpath jrt file > /Users/me/dev/jdks/openjdk_11.0.4_34.6_x64/lib/jrt-fs.jar > != old location: Classpath jrt file > /Users/me/dev/jdks/openjdk_11.0.4_34.6_x64/lib/jrt-fs.jar The "new location" ClasspathLocation object has module updates, i.e. add-exports. The "old location" ClasspathLocations object does not. "old location" maps to the last build state. It looks like somewhere is a bug (or race condition?) that the module updates are not properly restored in the build state.
