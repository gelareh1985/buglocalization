package demobug; public class SS_Default { void foo(Day day) { switch (day) { case SATURDAY, SUNDAY -> System.out.println("Weekend"); case MONDAY, TUESDAY, WEDNESDAY -> System.out.println("Weekday"); } switch (day) { case SATURDAY: System.out.println("Weekend"); case MONDAY: System.out.println("Weekday"); } } } enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY; } ------------- - In the above example, press Ctrl+Shift+F on all the code. No change is done with default formatter settings. - Use Add 'default' case quick fix on both the switch expressions. It results in the following. Note that the formatting of the first default is incorrect (no space before arrow and throw in newline even though syso in other cases are in same line). void foo(Day day) { switch (day) { case SATURDAY, SUNDAY -> System.out.println("Weekend"); case MONDAY, TUESDAY, WEDNESDAY -> System.out.println("Weekday"); default-> throw new IllegalArgumentException("Unexpected value: " + day); } switch (day) { case SATURDAY: System.out.println("Weekend"); case MONDAY: System.out.println("Weekday"); default: break; } - Now, press Ctrl+Shift+F. The default case in the switch with -> is formatted correctly after invoking the formatter explicitly. AST Rewrite should use the current formatter settings while generating the text edits.
