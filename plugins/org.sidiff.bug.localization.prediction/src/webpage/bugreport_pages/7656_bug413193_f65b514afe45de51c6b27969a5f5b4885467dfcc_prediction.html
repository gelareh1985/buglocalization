Creation Time 2013-07-18T16:55:40Z
The existing FORMATTER_BLANK_LINES_BEFORE_FIRST_CLASS_BODY_DECLARATION is already used for all kinds of type declarations (not only classes). The grammar for enum types is quite special, since it consists of two sections (enum constants; enum body declarations). We currently use the cited option to insert blank lines in front of the second section (enum body declarations). I wouldn't add another option for the first section (enum constants), but just use the same option for the first section as well. In the UI, we should replace the word "class" by "type" everywhere on the "Blank Lines" tab.
