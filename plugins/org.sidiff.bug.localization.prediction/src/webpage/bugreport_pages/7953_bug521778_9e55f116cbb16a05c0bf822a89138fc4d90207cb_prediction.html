Creation Time 2017-09-02T03:07:07Z
In the following methods, the floating-point casts have a intentional effect: they reduce the precision of their argument, rounding it to the nearest representable floating-point value: static int intThruFloat(int x) { return (int)(float)x; } static long longThruFloat(long x) { return (long)(float)x; } static long longThruDouble(long x) { return (long)(double)x; } However, if the save action "Remove unnecessary casts" is enabled, the casts are silently removed when I save the file. On a subsequent save, it deletes the other casts too! Each method is turned into "return x;".
