Creation Time 2019-02-12T07:17:16Z
Since Java 9 (http://openjdk.java.net/jeps/225) javadoc tool supports {@index XXX} tags to hint indexing. There is no proposal in Java editor autocomplete for this.
Creation Time 2019-02-12T16:27:00Z
Looking at https://docs.oracle.com/javase/10/docs/specs/doc-comment-spec.html we are missing autocomplete for the following. Java 9 added tags: * @index * @hidden * @provides * @uses Java 10 added tag: @summary
Creation Time 2019-02-12T16:58:37Z
Latest one: https://docs.oracle.com/en/java/javase/11/docs/specs/doc-comment-spec.html (thanks Alex!).
Creation Time 2019-02-26T12:26:08Z
(In reply to Alexander Kurtakov from comment #1) > * @provides > * @uses Bug 543996 has added some support for these two tags, but I also see room for improvement: For me completion of the tag only works at the start of a javadoc comment. After "@uses ISomeIfc explanation" the next "@" automatically expands to "@author stephan" No completion support is given for the mandatory argument to both service tags. No validation is given if a tag lacks details, moreover a correctly specified service type gets complaints from the spell-checker which should have no business spellchecking java identifiers :) I stumbled upon this when I noticed the additions in JavadocTagConstants, which have not consistently been integrated into TAG_NAMES and JAVADOC_TAG_TYPE.
Creation Time 2019-02-26T12:43:08Z
We don't get any proposal from ICodeAssist.codeComplete(...) for the mentioned tags. Moving to JDT Core.
