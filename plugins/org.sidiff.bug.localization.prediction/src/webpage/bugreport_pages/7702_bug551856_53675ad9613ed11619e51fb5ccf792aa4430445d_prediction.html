AbstractJavaModelTests has overloaded methods assertMarkers(..). More than once I have wasted time because I used the wrong overload, which only checks for build problems, not compile errors. In bug 550410 I started disambiguation by renaming one method to assertBuildPathMarkers and adding a temporary, deprecated delegation method. Since this caused warnings in the official build, the rename should be completed, i.e., all callers migrate to the new name.
