From the code: // The issue appears to be caused by ClassC.NestedClassC. Extracting it into its own // top level class causes build errors to disappear. It suffices to declare ClassC and NestedClassC as static - and adjust references ClassC.NestedClassC to ClassC.NestedClassC. This change removes the dependency from NestedClassC on the type parameter of its enclosing ClassC, i.e., NestedClassC is no longer generic. I further reduced the example to: //--- public class Bug { public void execute() { foo(bar(new ExampleType())); } public void foo(ClassB a) {} public ClassC bar(T t) { return null; } public class ClassC extends ClassB.NestedClassC> { public class NestedClassC {} } public abstract static class ClassB {} public class ExampleType {} } //--- to yield: ---------- 1. ERROR in /tmp/Bug.java (at line 4) foo(bar(new ExampleType())); ^^^ The method foo(Bug.ClassB) in the type Bug is not applicable for the arguments (Bug.ClassC) ---------- 2. ERROR in /tmp/Bug.java (at line 4) foo(bar(new ExampleType())); ^^^^^^^^^^^^^^^^^^^^^^ Type mismatch: cannot convert from Bug.ClassC to Bug.ClassB ---------- In order to accept, type inference would have to instantiate VC to ClassB.NestedClassC>. Not sure why it fails.
