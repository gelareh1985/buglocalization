Creation Time 2019-05-12T10:44:44Z
In bug 546797 I need API to ask if a module is a system module. Following the style of IMD#isAutoModule() we may want to add #isSystemModule(). Before doing so we may want to briefly check if more similar queries are relevant. Binary/Source is already covered by fetching the parent (either IModularClassFile or ICompilationUnit). I haven't yet given much thought to upgradeable modules and the upgrade-module-path. I was briefly thinking of introducing #getModuleKind() with a set of constants (an enum?) but I believe these things are well served with a simple set of #isX() API. To unblock bug 546797 I will start be exposing this functionality via the provisional JavaModelAccess.
